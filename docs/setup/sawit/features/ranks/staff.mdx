# Staff

Sawit staff ranks are handled through `LuckPerms`.

## Verified Staff Groups

| Group | System | Notes |
| --- | --- | --- |
| `trialhelper` | `LuckPerms` | Entry-level helper/moderation access. |
| `trialmod` | `LuckPerms` | Trial moderator layer above `trialhelper`. |
| `helper` | `LuckPerms` | Helper rank with higher vanish visibility and exemptions. |
| `mod` | `LuckPerms` | Moderator rank with rollback and vanish interaction access. |
| `admin` | `LuckPerms` | Admin rank with high-level moderation and admin permissions. |
| `coowner` | `LuckPerms` | Senior management rank above `admin`. |
| `owner` | `LuckPerms` | Highest verified staff rank in the package. |

These groups are visible in the LuckPerms storage as created groups, so they are not guesses.

| Item | Detail |
| --- | --- |
| High weights | `owner` uses weight `1000`, `coowner` uses `999`, `admin` uses `998`, `mod` uses `997`, `helper` uses `996`, `trialmod` uses `995`, and `trialhelper` uses `994`. |
| Command hider | The storage also contains role-specific command-hider visibility nodes for `admin`, `mod`, `helper`, `trialmod`, and `trialhelper`. |

| Coverage | The permission table has been refreshed against the provided LuckPerms web editor, so it is more complete than the earlier storage-only pass. |
