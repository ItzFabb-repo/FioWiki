# Premium

Sawit premium-style paid or perk ranks are handled through `LuckPerms`.

## Verified Premium Groups

| Group | Type | Notes |
| --- | --- | --- |
| `default` | Base | Starting player group. |
| `honey` | Premium | First verified premium-style group. |
| `apple` | Premium | Higher than `honey`. |
| `melon` | Premium | Higher than `apple`. |
| `cookie` | Premium | Higher than `melon`. |
| `amethyst` | Premium | Highest verified premium-style group in the package snapshot. |

## Notes On Accuracy

| Item | Detail |
| --- | --- |
| Direct vs inherited | This table focuses on permissions directly visible on each premium group in storage. Higher ranks may also inherit lower-rank permissions through the parent chain. |
| Excluded ambiguous nodes | I intentionally excluded nodes that appear to have been later removed or unset in the same storage snapshot. |
| Premium system split | These premium permissions belong to the LuckPerms ladder, while `VanguardRanks` is used for player league progression. |
| Editor check | The direct permission list has now been cross-checked against the provided LuckPerms web editor link. |

## Interpretation

| Point | Detail |
| --- | --- |
| Verified ladder | `honey`, `apple`, `melon`, `cookie`, and `amethyst` are the premium ladder names actually visible in the Sawit package. |
| Highest verified tier | `amethyst` is the highest verified premium group in this storage snapshot. |
| Accuracy limit | The hierarchy and the listed direct permissions are storage-verified, but a full clean LuckPerms export would still be better if you want a perfect inherited-permission matrix. |

:::info[Accuracy Note]
This page intentionally does not describe `VanguardRanks` as premium, because the Sawit package shows `VanguardRanks` being used for league progression instead.
:::
