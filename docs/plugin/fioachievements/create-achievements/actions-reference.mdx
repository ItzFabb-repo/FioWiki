# Actions Reference

This page lists the action names you can use inside `achievementTasks.required` and `achievementTasks.while`.

:::info[How to read this page]
`required` actions add progress. `while` entries do not add progress; they only decide whether a required progress event is allowed to count.
:::

## required: []

Use `required` for the action that should increase achievement progress.

```yaml
achievementTasks:
  required:
    - "BREAK_BLOCK;DIAMOND_ORE,DEEPSLATE_DIAMOND_ORE;10"
  while: []
```

## while: []

Use `while` for live conditions that must be true while the required action happens.

```yaml
achievementTasks:
  required:
    - "BREAK_BLOCK;DIAMOND_ORE;1"
  while:
    - "HOLD;DIAMOND_PICKAXE"
    - "WORLD;WORLD"
    - "HEALTH;10"
```

| Condition | Format | Meaning |
| --- | --- | --- |
| `HEALTH` | `HEALTH;<MAX_HEALTH>` | Player health must be at or below the value. |
| `WEAR` | `WEAR;<MATERIALS>;<SLOT>` | Player must wear matching armor in `HELMET`, `CHEST`, `LEGS`, or `BOOTS`. |
| `HOLD` | `HOLD;<MATERIALS>` | Player must hold one of the listed materials. |
| `HOLDING` | `HOLDING;<MATERIALS>` | Alias for `HOLD`. |
| `WORLD` | `WORLD;<WORLD_NAMES>` | Player must be in one of the listed worlds. |
| `BIOME` | `BIOME;<BIOME_NAMES>` | Player must be in one of the listed biomes. |
| `WEATHER` | `WEATHER;<CLEAR\|RAIN\|THUNDER>` | Player world weather must match. |
| `PHYSICAL` | `PHYSICAL;<ACTION>` | Player must match a movement state such as `SNEAK`, `SPRINT`, `SWIM`, `CLIMB`, `GLIDE`, `FALL`, or `JUMP`. |
| `PHYSICAL_ENTITY` | `PHYSICAL_ENTITY;<ENTITY>;<ACTION>` | Player must match a vehicle or entity movement state. |
| `PHYSICAL_BIOME` | `PHYSICAL_BIOME;<BIOMES>;<ACTION>` | Player must match a biome physical condition. |
| `SERVER` | `SERVER;<ACTION>` | Server/player lifecycle state must match where supported. |
| `ITEM` | `ITEM;<ACTION>;<MATERIALS>` | Player item state must match where supported. |
| `STEP_ON_BLOCK` | `STEP_ON_BLOCK;<MATERIALS>` | Player must stand on a matching block. |
| `BUCKET` | `BUCKET;<ACTION>;<MATERIALS>` | Player bucket state must match where supported. |
| `FISHING` | `FISHING;<ACTION>;<VALUE>` | Fishing state must match where supported. |
| `GAIN` | `GAIN;<ACTION>` | Gain state must match where supported. |
| Same action as `required` | Example: `BREAK_BLOCK;DIAMOND_ORE` | The condition must match the required action type and target. |
| Placeholder rule name | Example: `PLAYER_SATURATION` | Matches a named entry from `placeholderRules`. |

## Placeholder Rules For while

Use `placeholderRules` when a `while` condition must compare PlaceholderAPI output.

```yaml
achievementTasks:
  required:
    - "BREAK_BLOCK;DIAMOND_ORE;1"
  while:
    - "PLAYER_SATURATION"
placeholderRules:
  1:
    name: "PLAYER_SATURATION"
    type: "GREATHER THAN OR EQUALS"
    input: "%player_saturation%"
    output: "10"
```

| Option | Description |
| --- | --- |
| `name` | Condition name used inside `while`. |
| `type` | Comparison type used by the placeholder matcher. |
| `input` | PlaceholderAPI value to read. |
| `output` | Expected comparison value. |
