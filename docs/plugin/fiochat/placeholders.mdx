# Placeholders

FioChat currently uses more than one placeholder style. The most important one is its own PlaceholderAPI expansion under the `fiochat` identifier, but several module configs also define token-based placeholders in YAML.

## Placeholder systems in FioChat

The current source uses these practical placeholder styles:

1. Native PlaceholderAPI expansion values

```text
%fiochat_*%
```

2. PlaceholderAPI passthrough through the FioChat namespace

```text
%fiochat_papi_<placeholderapi_node>%
```

3. Module token placeholders defined in YAML

```text
{discord}
{vote}
{store}
```

## Dynamic placeholder families

### Channel statistics

The current source explicitly supports dynamic channel placeholders:

```text
%fiochat_channel_<id>_online%
%fiochat_channel_<id>_offline%
%fiochat_channel_<id>_messages_<window>%
%fiochat_channel_<id>_chatters_<window>%
```

Examples:

```text
%fiochat_channel_global_online%
%fiochat_channel_staff_messages_5m%
%fiochat_channel_trade_chatters_1h%
```

The source comments and parser support windows such as:

```text

all
```

## Placeholder behavior notes

The current placeholder expansion includes a few behavior rules that matter in practice:

- `*_toggle` keys are normalized to `*_state` for backwards compatibility
- dynamic chat color placeholders are resolved separately
- redeem-code placeholders are resolved separately
- command-scheduler placeholders are resolved separately
- PlaceholderAPI passthrough aliases are resolved before the normal FioChat key map

## Placeholder alias and replace support

The display-tag ecosystem also supports placeholder aliasing and replacement maps in its own config.

Example alias structure:

```yaml
settings:
  placeholder:
    aliases:
      - 'fio_thing': "%fiochat_antibot_verified%%fiochat_afk_state% %fiochat_vanish_state%"
```

Example replace map:

```yaml
settings:
  placeholder:
    replace:
      '%fiochat_afk_state%':
        'true': ' &7(AFK)'
        'false': ''
```

## Module token placeholders

Some module configs use token placeholders with `{...}` syntax instead of `%fiochat_*%`.

Examples come from files such as:

- `modules/announce/Server Announcements/placeholder.yml`
- `modules/announce/Message of the Day/placeholder.yml`

These are module-defined tokens, not the same system as PlaceholderAPI expansion values.

## Practical examples

```yaml
lines:
  - "Platform: %fiochat_user_platform%"
  - "Vanish: %fiochat_vanish_state%"
  - "AFK: %fiochat_afk_state%"
  - "Mentions: %fiochat_mentions_state%"
  - "PM: %fiochat_message_state%"
```

```yaml
lines:
  - "Global online: %fiochat_channel_global_online%"
  - "Global msgs (5m): %fiochat_channel_global_messages_5m%"
  - "Staff chatters (15m): %fiochat_channel_staff_chatters_15m%"
```
