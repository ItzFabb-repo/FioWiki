# Voucher YAML List

File voucher bawaan berada di:

```text
plugins/FioVoucher/vouchers/*.yml
```

Setiap file memakai nilai `voucher.id` sebagai kode voucher. Kode ini yang dipakai di command seperti `/voucher redeem <code>`, `/voucher admin give <player> <voucherid> [amount]`, `/voucher info <code>`, dan `/voucher admin freeze <voucherid>`.

## Usage cepat

```text
/voucher list
/voucher info money
/voucher redeem money
/voucher admin give Steve money 1
/voucher admin giveall trial_vip 1
/voucher admin freeze behavior_glow_unbreakable
/voucher admin unfreeze behavior_glow_unbreakable
```

:::tip
Saat membuat voucher baru, copy salah satu YAML di folder `vouchers`, rename file-nya, lalu pastikan `voucher.id` ikut diganti. Gunakan `voucher.id`, bukan nama file, untuk command.
:::

## Voucher bawaan

| File YAML | Voucher ID | Material | Usage |
| --- | --- | --- | --- |
| `example.yml` | `example` | `PLAYER_HEAD` | Template lengkap untuk contoh armor trims, command actions, toast, title, item reward, exp, bundle, enchantment, durability, unbreakable, dan permission gates. Cocok sebagai file awal saat membuat voucher baru. |
| `money.yml` | `money` | `PAPER` | Voucher ekonomi sederhana. Dipakai untuk memberi uang lewat action `[CONSOLE]`, biasanya terhubung ke plugin economy seperti Vault/Economy command. |
| `permission_kit.yml` | `permission_kit` | `CHEST` | Contoh voucher kit berbasis permission. Berguna untuk reward yang hanya bisa diklaim jika player memenuhi atau belum memiliki permission tertentu. |
| `random_food.yml` | `random_food` | `COOKED_BEEF` | Contoh voucher reward random makanan. Memakai `commands.random-claimed` untuk memilih salah satu reward berdasarkan chance. |
| `trial_vip.yml` | `trial_vip` | `PLAYER_HEAD` | Contoh voucher trial rank VIP. Dipakai untuk akses sementara dengan `settings.trial`, termasuk durasi trial dan opsi pause/stack bila diaktifkan. |
| `world_crate.yml` | `world_crate` | `TRIPWIRE_HOOK` | Contoh voucher crate yang dibatasi world. Cocok untuk reward yang hanya boleh diklaim di world tertentu melalui `settings.redeem.world`. |
| `behavior_banner_patterns.yml` | `behavior_banner_patterns` | `WHITE_BANNER` | Contoh item voucher dengan banner patterns. Dipakai untuk menguji metadata pattern pada banner. |
| `behavior_bundle_container.yml` | `behavior_bundle_container` | `BUNDLE` | Contoh bundle yang menampilkan item di dalam bundle melalui behavior `bundle_container`. Item di dalamnya dikunci sebagai tampilan reward. |
| `behavior_crossbow_model.yml` | `behavior_crossbow_model` | `CROSSBOW` | Contoh custom model data pada crossbow. Berguna untuk resource pack item model. |

| `behavior_leather_boots.yml` | `behavior_leather_boots` | `LEATHER_BOOTS` | Contoh pewarnaan leather boots. Berguna untuk reward armor leather dengan warna custom. |
| `behavior_leather_chestplate.yml` | `behavior_leather_chestplate` | `LEATHER_CHESTPLATE` | Contoh pewarnaan leather chestplate. Cocok untuk set kosmetik leather armor. |
| `behavior_leather_helmet.yml` | `behavior_leather_helmet` | `LEATHER_HELMET` | Contoh pewarnaan leather helmet. Dipakai untuk variasi warna item armor leather. |
| `behavior_leather_horse_armor.yml` | `behavior_leather_horse_armor` | `LEATHER_HORSE_ARMOR` | Contoh pewarnaan leather horse armor. Berguna untuk reward cosmetic mount. |
| `behavior_leather_leggings.yml` | `behavior_leather_leggings` | `LEATHER_LEGGINGS` | Contoh pewarnaan leather leggings. Dipakai untuk melengkapi set leather armor. |
| `behavior_model_data.yml` | `behavior_model_data` | `PAPER` | Contoh `model-data` pada item voucher. Cocok untuk server dengan resource pack custom item. |

## Pola command per voucher

Ganti `<voucherid>` dengan salah satu ID dari tabel di atas:

```text
/voucher redeem <voucherid>
/voucher info <voucherid>
/voucher delete <voucherid>
/voucher pause <voucherid>
/voucher admin give <player> <voucherid> [amount]
/voucher admin giveall <voucherid> [amount]
/voucher admin freeze <voucherid>
/voucher admin unfreeze <voucherid>
```

`/voucher pause <voucherid>` hanya bekerja untuk voucher trial yang mengaktifkan `settings.trial.enabled` dan `settings.trial.allow-pause`.
