# Developer API
FioChat provides a public API for integrations.

---

# Getting the API Instance
```java
import com.itzfabb.fiochat.api.FioChatApi;
import com.itzfabb.fiochat.api.FioChatApiProvider;

FioChatApi api = FioChatApiProvider.get();
if (api != null) {
    // safe to use
}
```

For Join/Quit-specific integrations:
```java
import com.itzfabb.fiochat.api.JoinQuitApi;
import com.itzfabb.fiochat.api.FioChatApiProvider;

JoinQuitApi joinQuitApi = FioChatApiProvider.getJoinQuitApi();
```

---

# Core API Capabilities
:::info[High level]
The exposed API focuses on player-facing chat systems, feature toggles, and selected module integration points.
:::

Common integration areas:
- Mention and private messaging behavior
- Join/Quit module integration
- Channel and audience behavior
- UX that respects player preferences and toggle state

---

# Recommended Integration Pattern
```java
public void runIntegration(Player player) {
    FioChatApi api = FioChatApiProvider.get();
    if (api == null) {
        return;
    }

    // Example: call available API methods in your plugin flow
    // (exact methods depend on the FioChat version you compile against)
}
```

---

# Soft-Depend Recommendation
In your plugin's `plugin.yml`:
```yaml
softdepend:
  - FioChat
```

Then resolve the API in `onEnable()` after server startup.

---

# Version Safety
:::warning[Compatibility]
Always compile against the same major/minor API version that you deploy.
If you target multiple FioChat builds, null-check the provider and fail gracefully.
:::
