# Placeholder
:::info[Scope]
This page is a full operational reference for configuring placeholders in FioChat.
It focuses on **how server owners use placeholders in YAML configs**.
:::

---

# Placeholder Systems in FioChat
FioChat uses three layers:

1. Native FioChat placeholders:
```yaml
%fiochat_*%
```

2. PlaceholderAPI passthrough and alias resolution.

3. Module token placeholders in YAML:
```yaml
{discord}
{vote}
{store}
{link}
```

---

# Where Placeholders Are Commonly Used
## Primary files
```yaml
modules/display-tag/display_tag.yml
modules/formats/config.yml
modules/announcements/announce.yml
modules/announcements/placeholder.yml
modules/player-motd/motd.yml
modules/player-motd/placeholder.yml
modules/server-motd/config.yml
```

:::tip[Rule]
- `%...%` usually means expansion placeholder.
- `{...}` usually means module token placeholder defined in a module `placeholder.yml`.
:::

# Placeholder Alias Integration (Display Tag)
In `modules/display-tag/display_tag.yml`:

```yaml
settings:
  placeholder:
    aliases:
      - 'fio_thing': "%fiochat_antibot_verified%%fiochat_afk_state% %fiochat_vanish_state%"
```

Usage in text templates:
```yaml
%fiochat_custom_fio_thing%
```

:::tip[Why use aliases]
Aliases reduce repeated long chains and make layouts easier to maintain.
:::

---

# Replace Map Integration (Display Tag)
Also in `display_tag.yml`:

```yaml
settings:
  placeholder:
    replace:
      '%fiochat_afk_state%':
        'true': ' &7(?)'
        'false': ''
```

This allows output decoration without changing source placeholder logic.

# PlaceholderAPI Passthrough and Alias Expansion
The source includes alias expansions for display modules (`tab`, `scoreboard`, `bossbar`) and `papi_...` passthrough patterns.

General behavior pattern:
```yaml
%fiochat_papi_<some_placeholderapi_node>%
```

Use only when the target expansion exists and is installed.

:::warning[Dependency Risk]
If a third-party PlaceholderAPI expansion is missing, output may be empty or unresolved.
:::

# Module Token Placeholders (`{...}`)
These are not `%...%` expansions. They are YAML token definitions.

## Announcements tokens
File: `modules/announcements/placeholder.yml`

Example:
```yaml
placeholders:
  vote:
    format: "<smallfont:vote>"
    hover:
      lines:
        - "<white>Click to open the vote link.</white>"
    action:
      open_url:
        - "https://example.com/vote"
```

Used in `modules/announcements/announce.yml`:
```yaml
message:
  - "Use {vote} and join {discord}!"
```

## Player MOTD tokens
File: `modules/player-motd/placeholder.yml`

Example:
```yaml
placeholders:
  discord:
    format: "<smallfont:discord>"
    hover:
      lines:
        - "<white>Click to visit our Discord.</white>"
    action:
      open_url:
        - "https://discord.gg/example"
```

Used in `modules/player-motd/motd.yml`:
```yaml
messages:
  - "Welcome to the server {discord} {store}"
```

Supported action types:
```yaml
run_command
suggest_command
copy_to_clipboard
open_url
```

:::note[Channel Behavior]
Hover/click behavior is primarily meaningful in chat-line rendering. Non-chat channels can still render text tokens but may not keep full click semantics.
:::
